# T-0554 USER 수거신청 3단계 스텝퍼/사진 표준화

## 1) 모바일 신청 플로우
- Request 탭 진입 후 `수거 요청 시작` 버튼을 누르면 3단계 신청 화면이 시작된다.
- 단계 구성:
  - 1단계: `수거 품목 선택`, `비닐 배송요청`
  - 2단계: `특이사항`, `참고사진`, `요청사항`
  - 3단계: `방문일정`, `주소`, `신청정보`, `결제수단`, `유의사항`
- 단계별 검증:
  - 1단계: 수거 품목 1개 이상 선택 필수
  - 3단계: 대표주소지/서비스가능여부/본인인증/유의사항 동의 검증 후 제출

## 2) 데이터 계약 정리(서버/모바일)
- 기존 요청 필드 유지:
  - `address`, `note`, `disposalItems`, `bagCount`
- 추가 필드:
  - `referencePhotoUrls: string[]` (선택)
- 저장 정책:
  - `referencePhotoUrls`는 `waste_photos`에 `type=REFERENCE`로 저장
  - `note`는 요청사항/특이사항/방문일정 요약 텍스트 저장

## 3) 사진 썸네일 표준화
- USER 신청 2단계 참고사진:
  - 업로드 즉시 썸네일 그리드 표시
  - 삭제 가능
- USER 상세:
  - `참고 사진(REFERENCE)`과 `기사 사진(기타)` 분리 썸네일 표시
  - 썸네일 탭 시 원본 확대 보기
- DRIVER 측정 화면:
  - 업로드 목록을 URL 텍스트 대신 썸네일 카드로 표시
  - 썸네일 탭 시 원본 확대 보기

## 4) 호환 정책
- 기존 단일 요청 API 인터페이스(`/waste-requests`)를 그대로 사용한다.
- `referencePhotoUrls`는 선택 필드이므로 기존 클라이언트 요청과 호환된다.
