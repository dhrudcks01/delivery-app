# 모바일 관리자 화면 스크롤/상태 점검 체크리스트 (T-0522)

## 목적
- OPS_ADMIN/SYS_ADMIN 화면에서 스크롤 불가, 상태 확인 어려움 문제를 재현/점검/검증하기 위한 수동 시나리오를 정의한다.

## 공통 사전조건
- 앱 로그인 완료
- 하단 탭 `신청` 또는 `이용내역`에서 관리자 화면 진입 가능 계정 사용
  - OPS_ADMIN 전용 계정
  - SYS_ADMIN 전용 계정
  - 멀티 권한 계정(예: DRIVER + OPS_ADMIN)

## 시나리오 1: OPS_ADMIN 화면 스크롤
1. OPS_ADMIN 계정으로 로그인한다.
2. 하단 탭 `신청` 진입 후 화면 최하단(로그아웃 버튼)까지 스크롤한다.
3. 다시 상단으로 스크롤한다.
4. 텍스트 입력(상태 필터, 기사 ID) 후에도 스크롤이 유지되는지 확인한다.

기대결과:
- 상/하단 모두 끊김 없이 스크롤 가능
- 키보드가 올라와도 입력 후 다른 카드 영역으로 스크롤 가능

## 시나리오 2: SYS_ADMIN 화면 스크롤
1. SYS_ADMIN 계정으로 로그인한다.
2. `OPS_ADMIN 신청`, `SYS_ADMIN 신청`, `권한 부여 대상 검색` 카드가 모두 보이도록 끝까지 스크롤한다.
3. 검색어 입력 후 키보드가 열린 상태에서 하단 카드로 스크롤한다.

기대결과:
- 긴 화면에서 카드 전체를 연속적으로 확인 가능
- 키보드 오픈 상태에서도 스크롤 잠김 없이 동작

## 시나리오 3: 관리자 화면 상태 점검 정보 확인
1. OPS_ADMIN/SYS_ADMIN 화면 상단의 `화면 상태 점검` 카드를 확인한다.
2. 목록 새로고침 버튼을 눌러 로딩 상태 텍스트가 갱신되는지 확인한다.

기대결과:
- 로딩/선택 상태를 화면 상단에서 즉시 확인 가능
- 상태 텍스트가 현재 화면 동작과 일치

## 시나리오 4: 멀티 권한 계정 점검
1. 멀티 권한 계정으로 로그인한다.
2. 홈 또는 내정보에서 역할 전환 후 `신청` 탭으로 이동한다.
3. OPS_ADMIN/SYS_ADMIN 화면에서 각각 스크롤이 정상 동작하는지 확인한다.

기대결과:
- 역할 전환 후에도 스크롤 문제 없이 동일한 동작 품질 유지
