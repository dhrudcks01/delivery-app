# T-0526 멀티 권한 최고권한 자동 진입 + USER 탭/내정보 정책 재정렬 체크리스트

## 점검 대상
- 앱 로그인 후 하단 탭(`홈/신청/이용내역/내정보`)
- 역할 조합 계정: USER / DRIVER / OPS_ADMIN / SYS_ADMIN / DRIVER+OPS_ADMIN / OPS_ADMIN+SYS_ADMIN
- 플랫폼: Android/iOS (Expo)

## 시나리오 1: 멀티 권한 최고 권한 자동 적용
1. `DRIVER+OPS_ADMIN` 계정으로 로그인
2. 역할 선택 UI(칩/토글) 노출 여부 확인
3. `홈` 탭에서 OPS_ADMIN 화면이 기본 노출되는지 확인
4. 동일 방식으로 `OPS_ADMIN+SYS_ADMIN` 계정 로그인 후 SYS_ADMIN 화면 기본 노출 확인

기대 결과:
- 역할 선택 UI 없이 최고 권한 기준 화면이 자동 적용된다.
- 권한 우선순위는 `SYS_ADMIN > OPS_ADMIN > DRIVER > USER`로 동작한다.

## 시나리오 2: 신청 탭 기능 고정 확인
1. USER 권한이 있는 계정으로 로그인
2. `신청` 탭 진입
3. 수거 신청 생성 폼 외 결제/주소 등록 UI 노출 여부 확인

기대 결과:
- `신청` 탭은 수거 신청 생성 기능만 제공한다.
- 결제수단 관리/주소관리 UI는 `신청` 탭에 노출되지 않는다.

## 시나리오 3: 내정보 > 주소관리 동선 및 대표주소 자동 적용
1. `내정보 > 주소관리` 진입
2. 주소 2개 이상 등록
3. 대표 주소지를 변경
4. `신청` 탭으로 이동해 신청 생성 시 주소 자동 반영 확인

기대 결과:
- 주소 다중 등록/수정/삭제가 가능하다.
- 대표 주소지 변경이 즉시 반영된다.
- 신청 생성 시 대표 주소지가 자동 사용된다.

## 시나리오 4: 내정보 > 결제수단 관리 동선 확인
1. `내정보 > 결제수단 관리` 진입
2. 카드 등록/변경 버튼 선택
3. 등록 페이지 열림 및 상태 조회 동작 확인

기대 결과:
- 결제카드 등록 진입이 `내정보 > 결제수단 관리` 경로로 제공된다.
- 결제수단 상태 조회 및 등록 페이지 오픈이 정상 동작한다.

## 시나리오 5: 로그아웃 위치 정책 확인
1. `홈/신청/이용내역` 탭에서 로그아웃 버튼 존재 여부 확인
2. `내정보 > 설정` 진입
3. `로그아웃` 실행 후 로그인 화면 복귀 확인

기대 결과:
- 로그아웃은 `내정보 > 설정 > 로그아웃` 경로에서만 제공된다.
- 로그아웃 수행 시 인증 정보가 정리되고 로그인 화면으로 이동한다.
