# T-0528 모바일 선택/입력창 가시성 수동 검증

## 목적
- 입력/선택 UI 활성화 시 포커스된 필드가 키보드에 가려지지 않는지 확인한다.
- 입력 종료 후 화면 위치가 자연스럽게 유지되는지 확인한다.

## 공통 환경
- Android: 작은 화면 1종 + 큰 화면 1종
- iOS: 노치 기기 1종 + 비노치/구형 비율 1종
- 세로/가로 회전 각각 1회 이상 확인

## 화면별 시나리오
1. 로그인
- 이메일 입력 후 비밀번호 입력으로 이동
- 비밀번호 포커스 시 입력창과 로그인 버튼 가시성 확인
- 키보드 닫힘 후 레이아웃 깨짐 여부 확인

2. 회원가입
- 아이디/이름/비밀번호 순서로 입력
- 마지막 입력 필드 포커스 상태에서 버튼 노출 확인
- 입력 완료 후 스크롤 위치가 갑자기 초기화되지 않는지 확인

3. USER 수거 신청
- 연락처 입력 후 요청사항(멀티라인) 입력
- 요청사항 입력 중에도 입력창 전체가 보이는지 확인
- 키보드 닫힘 후 상세/목록 영역 전환 시 레이아웃 유지 확인

4. 주소관리
- 주소 검색어 입력 후 검색 실행
- 상세 주소 입력 시 포커스 필드 가림 여부 확인
- 검색 결과 선택/저장/취소 반복 시 스크롤 튐 여부 확인

5. 결제수단 관리
- 인라인 결제수단 선택 패널 열기/닫기
- 카드번호/유효기간/비밀번호 앞 2자리/생년월일(또는 사업자번호) 입력
- 마지막 필드 입력 시 버튼 가시성 및 닫힘 후 위치 유지 확인

## 완료 기준
- 위 시나리오에서 입력창 가림, 버튼 접근 불가, 레이아웃 점프가 재현되지 않아야 한다.
