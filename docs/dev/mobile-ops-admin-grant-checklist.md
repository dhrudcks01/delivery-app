# T-0525 OPS_ADMIN 권한 부여 UX 체크리스트

## 점검 대상
- 앱 로그인 후 `내정보 > 권한 신청/승인`
- 역할별 계정: DRIVER / OPS_ADMIN / SYS_ADMIN

## 시나리오 1: OPS_ADMIN 검색 기반 부여
1. OPS_ADMIN 계정으로 로그인 후 `내정보 > 권한 신청/승인` 진입
2. 검색어 입력 후 `검색` 버튼 선택
3. 후보 사용자 선택 후 `선택 사용자 권한 부여` 실행

기대 결과:
- 검색 결과는 `DRIVER 권한 보유 + OPS_ADMIN 미보유` 사용자만 노출된다.
- 권한 부여 성공 시 성공 메시지가 노출된다.
- 부여 후 새로 조회하면 부여된 사용자는 후보 목록에서 제외된다.

## 시나리오 2: DRIVER 메뉴 노출 차단
1. DRIVER 계정으로 로그인 후 `내정보 > 권한 신청/승인` 진입

기대 결과:
- OPS_ADMIN 부여/신청 메뉴가 노출되지 않는다.
- 안내 문구만 노출된다.

## 시나리오 3: 권한 없음 피드백
1. USER 또는 DRIVER 토큰으로 OPS_ADMIN 부여 API 호출 시도

기대 결과:
- 서버가 403으로 응답한다.
- 앱에서 권한 없음 에러 메시지를 확인할 수 있다.

## 시나리오 4: SYS_ADMIN 승인 진입 유지
1. SYS_ADMIN 계정으로 로그인 후 `내정보 > 권한 신청/승인` 진입
2. `권한 승인/부여 화면 열기` 선택

기대 결과:
- 기존 SYS_ADMIN 승인 화면으로 정상 이동한다.
