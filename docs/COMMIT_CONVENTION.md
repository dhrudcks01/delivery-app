# 커밋 메시지 규칙

형식:
[타입] : T-티켓번호 작업내용 요약

예시:
[chore] : T-0003 로컬 MySQL 개발 가이드 문서 작성
[feat]  : T-0103 이메일 회원가입 및 로그인 기능 구현
[fix]   : T-0404 토스 결제 승인 금액 검증 오류 수정
[docs]  : T-0001 프로젝트 README 구조 정리
[test]  : T-0601 인증 및 RBAC 통합 테스트 추가
[refactor] : T-0302 상태 전이 로직 리팩토링

타입 종류:
- chore    : 설정/초기화/환경
- feat     : 기능 추가
- fix      : 버그 수정
- docs     : 문서 변경
- test     : 테스트 추가/수정
- refactor : 기능 변화 없는 코드 개선
